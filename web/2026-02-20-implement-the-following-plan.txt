│                                                                              │
│ Contexto                                                                     │
│                                                                              │
│ Integrar un programa Solana de trazabilidad (en /traza) con una aplicación   │
│ Next.js (en /web) para rastrear productos a través de la cadena de           │
│ suministro. El sistema usa roles validados (Producer, Factory, Retailer,     │

│ wallet, solicitar un rol, obtener validación de la autoridad del programa, y │

│ - Wallet: @solana/wallet-adapter (Phantom, Solflare)                         │

│ Consideraciones Técnicas                                                     │
│                                                                              │
│ Transferencias                                                               │
│                                                                              │
│ - Solo se permiten transferencias completas (no parciales)                   │
│ - Rutas válidas: Producer→Factory, Factory→Retailer, Retailer→Consumer       │
│ - El token pasa a status InTransfer hasta que el receptor acepta             │
│                                                                              │
│ Factory Tokens                                                               │
│                                                                              │
│ - Factory DEBE incluir al menos 1 source_token de Producer al crear          │
│ - Source tokens se pasan en remainingAccounts (máx. 10)                      │
│                                                                              │
│ PDAs                                                                         │
│                                                                              │
│ - Todos los PDAs son deterministas con seeds específicos                     │
│ - Usar helpers de pda.ts en todas las instrucciones                          │
│ - No hardcodear cálculos de PDAs                                             │
│                                                                              │
│ Mints                                                                        │
│                                                                              │
│ - Cada TraceToken necesita un mint único (Keypair.generate())                │
│ - El mint NO es un SPL Token mint, solo un identificador                     │
│ - El mint debe ser signer en createToken                                     │
│                                                                              │
│ Queries Optimizadas                                                          │
│                                                                              │
│ - Usar memcmp filters en getProgramAccounts para filtrar por creator/owner   │
│ - Calcular offsets correctamente (discriminator 8 bytes + campos)            │
│                                                                              │
│ Verificación End-to-End                                                      │
│                                                                              │
│ 1. Setup inicial:                                                            │
│   - Conectar wallet (Phantom)                                                │
│   - Authority: Inicializar programa                                          │
│ 2. Flujo completo:                                                           │
│   - Usuario A: Registrar como Producer → Authority valida                    │
│   - Usuario B: Registrar como Factory → Authority valida                     │
│   - Usuario C: Registrar como Retailer → Authority valida                    │
│   - Usuario D: Registrar como Consumer → Authority valida                    │
│ 3. Crear y transferir:                                                       │

│   - Producer: Transferir a Factory                                           │

│   - Factory: Transferir a Retailer                                           │
│   - Retailer: Aceptar, transferir a Consumer                                 │
│   - Consumer: Aceptar                                                        │

│   - Verificar source_tokens en tokens de Factory                             │
